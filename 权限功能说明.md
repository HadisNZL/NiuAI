# NIU AI 权限管理功能

## 新增功能

### 1. 路径授权机制
- 默认只允许访问用户主目录（`~`）
- 访问其他路径时需要先授权

### 2. 命令说明

#### 授权路径
```
授权 /path/to/directory
```
例如：
- `授权 /Applications` - 授权访问 Applications 目录
- `授权 /System` - 授权访问 System 目录
- `授权 /` - 授权访问整个根目录

#### 查看已授权路径
```
已授权路径
授权列表
authorized
```

#### 进入目录（需要权限）
```
进入 /Applications
cd /System
切换到 /usr/local
```

如果目录未授权，系统会提示：
```
⚠️ 该路径需要授权访问：/Applications

该路径不在已授权范围内。如需访问，请输入：
授权 /Applications
```

### 4. 权限范围

- 授权是递归的：授权 `/usr` 后，可以访问 `/usr/local`、`/usr/bin` 等子目录
- 授权持续到服务重启，不会永久保存
- 系统级权限限制仍然生效（如 macOS 的 SIP 保护）

### 5. 安全建议

- 只授权必要的目录
- 避免直接授权根目录 `/`
- 注意某些系统目录即使授权也可能无法访问（需要 root 权限或 macOS 完全磁盘访问权限）
